<h2>DAY 1</h2>
- List Down : </br> </br>

1.) Functional Requirements Of System : </br>
- USER can PLAY & PAUSE song.
- USER can create PLAYLISTS, add SONGS to it and play entire playlist int SEQUENTIAL and RANDOM manner.
- SYSTEM should support MULTIPLE OUTPUT DEVICES like Bluetooth Speaker, Wired Earbuds etc. </br> </br>

2.) Non Functional Requirements Of System : </br>
- Entire Design should be Scalable (I mean of course 😐).
- New Features (like playing another song and current playing song should pause, can add new output device etc) should easily integrated (indirectly pointing to scalability 🤦‍♂️).

</br>
- Happy Flow : </br> </br>

- START
  - `USER` Interact With `APP`.
  - `USER` connect headphone (or any other output device).
  - Play a `SONG`.
  - Pause that `SONG`.
  - Create a `PLAYLIST`.
  - Add Songs In That `PLAYLIST`.
  - Play That Playlist In Random Way.
- END
